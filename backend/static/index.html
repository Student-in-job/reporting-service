Execute Report

Execute report SQL and return data in unified JSON format. The main method of the service. Accepts report ID (UUID) or slug.

POST /api/v1/reports/{report_id}/data viewer+
Path Parameters
ParameterTypeDescription
report_idUUID or stringReport UUID or slug
Request Body
FieldTypeRequiredDescription
date_fromdateyesPeriod start (YYYY-MM-DD)
date_todateyesPeriod end (YYYY-MM-DD)
filtersobjectnoAdditional filters as key-value pairs
Response (200) — Unified Format
FieldTypeDescription
report_idUUIDReport ID
slugstringReport slug
titlestringReport name
typestringVisualization type
columns[]arrayColumn definitions: key, label, type
data[]arrayData rows (keys match columns[].key)
totalsobject?Aggregated totals (optional)
metaobjectgenerated_at, execution_time_ms, total_rows, filters_applied
Request Example
Request{
  "date_from": "2026-04-01",
  "date_to": "2026-04-27",
  "filters": {
    "scoring_type": "INITIAL_SCORING"
  }
}
Response Example
Response 200{
  "report_id": "b6680d71-4afc-4937-9148-50f9f319f736",
  "slug": "scoring-by-day",
  "title": "Скоринг по дням",
  "type": "bar_chart",
  "columns": [
    { "key": "d", "label": "Дата", "type": "date" },
    { "key": "c", "label": "Уникальных клиентов", "type": "number" }
  ],
  "data": [
    { "d": "2026-04-01", "c": 5 },
    { "d": "2026-04-02", "c": 9 },
    { "d": "2026-04-03", "c": 15 },
    { "d": "2026-04-04", "c": 8 },
    { "d": "2026-04-05", "c": 22 }
  ],
  "totals": null,
  "meta": {
    "generated_at": "2026-04-27T16:14:14.249537+00:00",
    "execution_time_ms": 1,
    "total_rows": 26,
    "filters_applied": {
      "date_from": "2026-04-01",
      "date_to": "2026-04-27",
      "scoring_type": "INITIAL_SCORING"
    }
  }
}
Errors
  • 400 Invalid filters (date_from > date_to, range exceeds max_range_days)
  • 404 Report not found or inactive
  • 504 SQL execution timeout (>30s)
5
Analyze SQL

Auto-detect column definitions from SQL query. Executes with LIMIT 0, reads PostgreSQL column metadata (name, type), and maps to Report Service types.

POST /api/v1/admin/reports/analyze-sql admin
Request Body
FieldTypeRequiredDescription
datasource_idUUIDyesTarget datasource
sql_querystringyesSQL query to analyze
Response (200)
FieldTypeDescription
columns[].keystringColumn name from SQL
columns[].labelstringAuto-generated label (title case)
columns[].typestringDetected type: string | number | date | datetime | boolean
Request Example
Request{
  "datasource_id": "f76d6370-b34e-4f6c-9dbb-92efd6858ba6",
  "sql_query": "SELECT DATE(created_at) d, COUNT(DISTINCT client_id) c FROM client_scoring_results WHERE scoring_type = :scoring_type AND created_at BETWEEN :date_from AND :date_to GROUP BY DATE(created_at) ORDER BY d"
}
Response Example
Response 200{
  "columns": [
    { "key": "d", "label": "D", "type": "date" },
    { "key": "c", "label": "C", "type": "number" }
  ]
}
Errors
  • 400 Invalid SQL or datasource connection error

7
Create Report

Create a new report. If config.columns is empty or missing, columns are auto-generated from SQL metadata.

POST /api/v1/admin/reports admin
Request Body
FieldTypeRequiredDescription
slugstringyesUnique URL slug
namestringyesDisplay name
descriptionstringnoDescription
groupstringnoMenu group
typestringnotable (default) | bar_chart | pie_chart | scatter_plot
datasource_idUUIDyesTarget datasource
sql_querystringyesSQL with :param placeholders
configobjectnofilters, columns (auto if empty), max_range_days
Request Example
Request{
  "slug": "scoring-by-day",
  "name": "Скоринг по дням",
  "description": "Уникальные клиенты, прошедшие INITIAL_SCORING",
  "group": "dashboard",
  "type": "bar_chart",
  "datasource_id": "f76d6370-b34e-4f6c-9dbb-92efd6858ba6",
  "sql_query": "SELECT DATE(created_at) d, COUNT(DISTINCT client_id) c FROM client_scoring_results WHERE scoring_type = :scoring_type AND created_at BETWEEN :date_from AND :date_to GROUP BY DATE(created_at) ORDER BY d",
  "config": {
    "filters": [
      { "name": "date_from", "type": "date", "required": true, "label": "Дата с" },
      { "name": "date_to", "type": "date", "required": true, "label": "Дата по" },
      { "name": "scoring_type", "type": "string", "required": true, "label": "Тип скоринга" }
    ],
    "max_range_days": 90
  }
}
Auto-generation: config.columns is omitted — the backend executes SQL with LIMIT 0, detects column types from PostgreSQL metadata, and fills columns automatically.
Response: 201 Created

Same structure as Get Report (Admin). Note auto-generated columns in config.

Errors
  • 400 Datasource not found
8
Update Report

Partial update — send only changed fields. If sql_query changes and columns not provided, columns are auto-regenerated.

PUT /api/v1/admin/reports/{report_id} admin
Request Example
Request{
  "config": {
    "columns": [
      { "key": "d", "type": "date", "label": "Дата" },
      { "key": "c", "type": "number", "label": "Уникальных клиентов" }
    ],
    "filters": [/* ... */],
    "max_range_days": 90
  }
}
Response: 200 OK

Full report object (same as Get Report).

Errors
  • 400 Datasource not found (if datasource_id changed)
  • 404 Report not found

Delete Report

Soft delete — sets is_active = false. Report disappears from public list but remains in database.

DELETE /api/v1/admin/reports/{report_id} admin
Response: 204 No Content
Response 204// Empty body
Errors
  • 404 Report not found

14
Delete Datasource

Delete datasource. Blocked if any reports (active or inactive) are linked to it.

DELETE /api/v1/admin/datasources/{datasource_id} admin
Response: 204 No Content
Response 204// Empty body
Error Example
Response 409{
  "detail": "Datasource has linked reports, delete them first"
}
Errors
  • 404 Datasource not found
  • 409 Has linked reports
